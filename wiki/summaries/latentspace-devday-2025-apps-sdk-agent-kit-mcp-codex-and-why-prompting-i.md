# DevDay 2025: Apps SDK, Agent Kit, MCP, Codex and Why Prompting Is More Important than Ever

- **Source URL**: https://www.latent.space/p/devday-2025-apps-sdk-agent-kit-mcp
- **Date**: October 7, 2025
- **Publication**: Latent Space

## Summary

At OpenAI DevDay 2025, Sherwin Wu (Head of Engineering, Platform) and Christina Huang walk through the launch of AgentKit — a suite spanning Agent SDK, visual Agent Builder, evals, and deployment tools — plus the Apps SDK that inverts the traditional website-chatbot paradigm by embedding third-party apps directly within ChatGPT. OpenAI's adoption of Anthropic's MCP protocol for universal tool connectivity is a notable concession that validates MCP as the emerging standard for agent-tool interop. The conversation reveals that prompting remains a critical and persistent skill (contrary to 2023-era predictions it would disappear), that OpenAI is dogfooding these tools with its own customer support powered by AgentKit, and that visual workflow builders are essential as agent systems grow to dozens of nodes — positioning OpenAI's agent platform directly against LangGraph, CrewAI, and similar orchestration frameworks.

## Key Claims

- OpenAI launched AgentKit: Agent SDK, visual Agent Builder, automated evals, and deployment tooling
- Apps SDK embeds third-party applications directly inside ChatGPT, inverting the app-chatbot paradigm
- OpenAI adopted Anthropic's MCP protocol for universal tool connectivity in AgentKit
- Christina Huang demo'd building a customer support agent in 8 minutes using the visual Agent Builder
- OpenAI's own customer support at openai.com is powered by AgentKit (internal dogfooding)
- Prompting remains a critical skill — contrary to predictions from 2023 that it would be automated away
- Automated prompt optimization is described as "zero-gradient fine-tuning"
- OpenAI is pursuing a Service Health Dashboard targeting five-nines reliability
- ChatKit launched as an embeddable, evergreen chat interface component
