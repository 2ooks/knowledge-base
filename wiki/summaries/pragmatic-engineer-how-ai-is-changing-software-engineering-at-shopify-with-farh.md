# How AI is Changing Software Engineering at Shopify with Farhan Thawar

**Source URL**: https://newsletter.pragmaticengineer.com/p/how-ai-is-changing-software-engineering
**Date**: July 2, 2025
**Publication**: The Pragmatic Engineer

## Summary

Shopify's Head of Engineering details the company's "all in on AI" strategy: unlimited token budgets for engineers, an internal LLM proxy for privacy and tracking, early GitHub Copilot access (first company outside GitHub to use it), and non-engineers using Cursor for vibe coding with PR submission requirements. Critically, Shopify requires coding interviews for director-level-and-above hires and encourages AI tool use during interviews, finding that candidates without copilots "get creamed" but that the best engineers still manually fix obvious issues rather than re-prompting. This is the most detailed public case study of enterprise AI-first engineering transformation with concrete policy choices.

## Key Claims

- Shopify was the first company outside GitHub to use GitHub Copilot
- Shopify places no limit on AI token spending for engineers
- Shopify built an internal LLM proxy for privacy protection, token tracking, and tool management
- Non-engineering teams at Shopify use Cursor for vibe coding, but must understand code to submit PRs
- Tobi Lutke's viral memo established "reflexive AI usage" as a baseline expectation at Shopify
- Directors and above must still pass a coding interview, which now includes AI tool usage
- Candidates without a copilot in interviews "usually get creamed by someone who does"
- Shopify is hiring 1,000 interns despite AI-first strategy -- AI-first is not about reducing headcount
- Shopify uses MCPs (Model Context Protocols) as part of their AI tool infrastructure
