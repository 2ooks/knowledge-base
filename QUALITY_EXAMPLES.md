## Too Generic (stop producing these)

**"Azure should launch vertical-model-training-as-a-service" (from top-5-strategic-insights.md)**
Azure ML/AI Foundry already pitches fine-tuning, distillation, RFT. You'd get bounced in a review for not knowing the existing portfolio. The genuinely novel angle was buried: GitHub's interaction corpus as a strategic asset for a GitHub-trained coding model tier. That should have been the whole insight, not a sub-bullet.

**"Should GitHub build vibe coding workflows?" (from top-5-strategic-insights.md)**
This is a product question, not a partnership thesis. Microsoft already has the answer in motion (Copilot Workspace, Spark). For BD docs, reframe around partnerships: who does GitHub acquire or invest in (Replit, Lovable, v0, Bolt), and what's the make-vs-buy logic?

## Structural Anti-Patterns

**Questions instead of recommendations.** Every insight ending with "Should GitHub…?" An exec asks "what are *you* recommending?" Pick a side.

**No partner names.** A partnership-positioning doc with zero proposed partners, investments, or term-sheet shapes. Name the company, the deal shape, the price range.

**5 equal insights.** Presenting 5 findings as equal weight when 2 are doing 80% of the work. Force-rank by actionability. Lead with the sharpest.

**Gaps at the bottom.** If a research gap is sharper than a finding, promote it to a top-level insight.
